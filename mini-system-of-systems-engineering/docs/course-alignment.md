# Course Alignment -- System of Systems Engineering

## MIT 16.842: Fundamentals of Systems Engineering
- **System boundaries**: SoS blur traditional system boundaries
- **Emergent behavior**: Properties not predictable from components alone
- Connection: `sos_emergence_measure` quantifies emergent complexity

## CMU 24-654: Systems Thinking
- **Governance models**: Directed, acknowledged, collaborative, virtual SoS
- **Interoperability**: Technical, semantic, organizational levels

## Stanford AA274: Multi-agent Systems
- **Decentralized control**: SoS control without central authority
- **Coordination mechanisms**: Consensus, auctions, market-based

## Key Concepts
```
SoS Types: Directed -> Acknowledged -> Collaborative -> Virtual
Maier's criteria: Operational independence, managerial independence,
                  evolutionary development, emergent behavior
Interoperability maturity: Level 0 (none) -> Level 5 (conceptualized)
```

## Exercises
1. Classify a real SoS (e.g., smart grid, air traffic control)
2. Measure interoperability between two systems
3. Design governance for a collaborative SoS
