# System of Systems Engineering Theory

## Maier's SoS Classification (1998)

| Type | Control | Goals | Example |
|------|---------|-------|---------|
| Directed | Centralized | Unified purpose | Integrated air defense |
| Acknowledged | Central coordination | Shared objectives | Joint force operations |
| Collaborative | Voluntary agreements | Collective benefit | Internet (IETF standards) |
| Virtual | No central authority | Emergent | Global supply chains |

## Five SoS Characteristics (Boardman & Sauser, 2006)

1. **Autonomy**: Constituent systems operate independently
2. **Belonging**: Systems choose to participate in the SoS
3. **Connectivity**: Interoperable interfaces enable interaction
4. **Diversity**: Heterogeneous systems increase robustness
5. **Emergence**: New capabilities arise from interactions

## Architecture Types

- **Hierarchical**: Tree structure, clear chain of command
- **Networked**: Peer-to-peer with redundant paths
- **Hub-and-Spoke**: Central coordinator, efficiency focused
- **Mesh**: Fully connected, maximum resilience
- **Hybrid**: Combinations of the above

## Interoperability (LISI Model)

| Level | Name | Description |
|-------|------|-------------|
| L0 | Isolated | No direct connection |
| L1 | Connected | Basic electronic link |
| L2 | Functional | Minimal common data |
| L3 | Domain | Shared data model |
| L4 | Enterprise | Universal interoperability |

## Emergence

Emergent behaviors are properties of the whole not present in any part.
- **Weak emergence**: Predictable from component rules (e.g., flocking)
- **Strong emergence**: Irreducible to parts (e.g., consciousness)
- **Nominal emergence**: Intended and beneficial
- **Pathological emergence**: Unintended and harmful
