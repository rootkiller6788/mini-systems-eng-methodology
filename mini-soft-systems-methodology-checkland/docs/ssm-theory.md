# Soft Systems Methodology (Checkland)

## 1. Hard vs Soft Systems

Peter Checkland developed SSM at Lancaster University (1970s) after finding
that hard systems engineering approaches failed when applied to messy human
situations.

| Hard Systems | Soft Systems |
|---|---|
| Problem is well-defined | Problem is ill-structured |
| Single objective | Multiple conflicting objectives |
| Optimization | Learning and accommodation |
| The system EXISTS | The system is a CONSTRUCT to explore |
| "How should we do it?" | "What should we do?" |

## 2. Weltanschauung (Worldview)

The central concept: every problem description embodies a **Weltanschauung**
(worldview). Different stakeholders with different W's will describe the
same situation differently. SSM makes these W's explicit.

SSM seeks **accommodation** — not consensus (everyone agrees) or compromise
(everyone gives up something), but a framework where conflicting W's can
coexist productively.

## 3. CATWOE Analysis

The formal mnemonic for formulating Root Definitions:
- **C**ustomers: Victims or beneficiaries of T
- **A**ctors: Those who would do T
- **T**ransformation: Input transformed to Output
- **W**eltanschauung: Worldview making T meaningful
- **O**wners: Those who could stop T
- **E**nvironment: Constraints taken as given

## 4. Root Definitions

"A system to do P by Q in order to achieve R"

5E evaluation criteria extend the classic 3E (Checkland & Scholes, 1990):
- E1 (Efficacy): Does T work?
- E2 (Efficiency): Minimal resource use?
- E3 (Effectiveness): Achieves higher purpose?
- E4 (Ethicality): Morally acceptable?
- E5 (Elegance): Aesthetically pleasing?

## 5. Conceptual Models

NOT models of the real world. Models of what the system MUST DO to BE the
system named in the Root Definition. Built from the minimum necessary
activities (verbs), structured by logical dependency.

## 6. The 7-Stage Learning Cycle

1. Problem situation considered problematic
2. Problem situation expressed (Rich Picture)
3. Root definitions of relevant systems
4. Conceptual models built
5. Comparison of models with real world
6. Desirable and feasible changes
7. Action to improve the situation

The cycle is never-ending. Each iteration deepens understanding.
